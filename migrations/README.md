# Database migrations (Alembic)

The DB schema is managed two ways:

- **Dev (SQLite):** the app calls `create_all` at startup, which creates any
  missing tables. Convenient, zero-step. It does NOT add new columns to existing
  tables, so for schema changes use the migrations below.
- **Prod / persistent DBs:** use Alembic. It tracks the schema version and
  applies incremental changes (including column additions and Postgres).

The database URL comes from `SPRITZ_DB_URL` (default `sqlite:///./spritz.db`),
shared by the app and Alembic, so set it once.

## Common commands

```bash
# apply all migrations to the configured DB
SPRITZ_DB_URL=... alembic upgrade head

# after changing a model in app/models.py, autogenerate a migration
# (run against a DB already at the previous head, or a clean DB for the first)
SPRITZ_DB_URL=... alembic revision --autogenerate -m "what changed"

# inspect / move
alembic current
alembic history
alembic downgrade -1
```

Review every autogenerated migration before committing: Alembic is good but not
perfect, especially for renames and data migrations.
